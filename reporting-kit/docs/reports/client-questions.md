# Отчёт «Вопросник заказчику»

Подготовка вопросов заказчику по открытым пунктам проекта для отправки в
Telegram. Адресат — нетехнический (со стороны Клиента).
Цель — задать пару вопросов так, чтобы человек легко понял и быстро ответил.

Ответ читает владелец сам и затем обновляет базу знаний и процесс.

- **Источник данных:** `docs/project-map.md`, раздел «Открытые вопросы» (статус «ждём») + «Ответственные за решения».
- **Формат:** Telegram plain text (без markdown), русский язык.
- **Объём:** 1—3 вопроса за раз — не вываливать всё сразу.

ПЕРЕД НАЧАЛОМ

Сначала прочитай актуальный файл docs/project-map.md (через GitHub MCP get_file_contents или Read), не отвечай по памяти. Возьми открытые вопросы из раздела «Открытые вопросы» (статус «ждём») и ответственных из раздела «Ответственные за решения». Если вопросов много — выбери 1—3 самых важных или спроси у пользователя, какие включить. Лучше пара вопросов, на которые реально ответят, чем длинный список.

ПРИНЦИПЫ ФОРМУЛИРОВКИ
• Вежливо, без давления и конфронтации. Это просьба помочь, а не претензия.
• Очень простым языком. Никакого технического жаргона, имён полей, кода, номеров задач.
• Не обсуждаем бюджеты, деньги и жёсткие сроки-обязательства.
• По каждому вопросу: предложи варианты понятными словами, кратко освети риск или последствие каждого, дай свою рекомендацию.
• Укажи, кто должен ответить.
• Один вопрос = один понятный выбор. Если в пункте спрятано несколько решений — раздели на отдельные вопросы.

ФОРМАТ ВЫВОДА — TELEGRAM PLAIN TEXT

Отчёт уходит в Telegram как plain text. Telegram не парсит markdown в обычных сообщениях.
• Без markdown: никаких звёздочек, подчёркиваний, бэктиков, решёток, угловых скобок-цитат и MarkdownV2-экранирования (\. \( и т.п.).
• Заголовки — КАПСОМ, перед каждым тематический эмодзи (📋 ❓ 💡 💬 и т.п.). Эмодзи только в начале заголовка. Не используй эмодзи 🙏.
• Разделы отбивай пустой строкой, без линий-разделителей.
• Списки — маркером • в начале пункта. Тире — только как знак препинания внутри предложения, не как маркер списка.
• Номера issue/PR не указывай. Технические токены — обычными словами.
• Язык — русский.
