# Отчёт «Состояние проекта»

Срез состояния проекта по [карте проекта](../project-map.md): где мы, что
сделано, что мешает запуску. Простым языком, для нетехнической аудитории
(заказчик, сообщество).

- **Источник данных:** `docs/project-map.md` (+ актуальные issues при необходимости).
- **Репозиторий:** `<owner>/<repo>`.
- **Формат:** Telegram plain text (без markdown), русский язык.
- **Тон:** просто и понятно, коротко, без жаргона.

---

## Промпт

Скопируй всё ниже этой строки.

---

Актуализируй и опиши состояние проекта по карте проекта (repo <owner>/<repo>, файл docs/project-map.md).

Сначала прочитай актуальный файл (через GitHub MCP get_file_contents или Read), не отвечай по памяти. Основной источник — docs/project-map.md; если нужны свежие данные сверх карты — проверь открытые issues репозитория.

Структура ответа (каждый блок — отдельный раздел):
- Цель
- Шаги
- Что сделано
- Что сейчас
- Что дальше
- Что потом
- Что мешает запуску
- Что после запуска

Всё простыми словами. Коротко и понятно.

ФОРМАТ ВЫВОДА — TELEGRAM PLAIN TEXT

Отчёт уходит в Telegram как plain text. Telegram не парсит markdown в обычных сообщениях — спецсимволы пойдут как мусор.

• Без markdown: никаких звёздочек, подчёркиваний, бэктиков, решёток, угловых скобок-цитат и экранирования.
• Заголовки разделов — КАПСОМ, перед каждым тематический эмодзи (🎯 🧭 ✅ ⏳ ➡️ 🔮 ⛔ 🚀 и т.п.). Эмодзи только в начале заголовка, не внутри текста. Не используй эмодзи 🙏.
• Разделы отбивай пустой строкой, без линий-разделителей.
• Списки — маркером • в начале пункта. Тире — только как знак препинания внутри предложения.
• Номера issue/PR не указывай. Технические токены — обычным текстом без бэктиков.
• Язык — русский.
