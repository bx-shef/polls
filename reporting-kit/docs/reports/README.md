# Отчёты по проекту

Шаблоны (промпты) для регулярных отчётов. Эти файлы — **канон** текста промпта.

Промпты уже доступны как Claude Code навыки и команды — `/report-status`,
`/report-digest`, `/report-questions` (см. [`.claude/skills/`](../../.claude/skills/README.md)).
Ручное копирование больше не обязательно; файлы здесь остаются как канон промпта
и резерв. Тела навыков — зеркало этих файлов, синхронность проверяет
`scripts/check-skills.sh`.

| Отчёт | Когда нужен | Файл |
|---|---|---|
| **Состояние проекта** | Срез «где мы и что дальше» по [карте проекта](../project-map.md). Для заказчика/сообщества, простым языком. | [project-status.md](project-status.md) |
| **Инженерный дайджест** | «Что сделано» по GitHub-репозиториям за период. Для Telegram, нетехническая аудитория. Две версии: кратко и подробно. | [engineering-digest.md](engineering-digest.md) |
| **Вопросник заказчику** | Задать клиенту 1—3 открытых вопроса (варианты, риски, рекомендация). Для Telegram, простым языком. | [client-questions.md](client-questions.md) |

> Как превратить эти отчёты в Claude Code skills (слэш-команды) — см. [skills-handoff.md](skills-handoff.md).

## Чем отличаются

- **Состояние проекта** отвечает на вопрос «куда мы идём и что мешает» — это
  взгляд от цели (запуск к сентябрю 2026). Источник — `docs/project-map.md`.
- **Инженерный дайджест** отвечает на вопрос «что конкретно сделано за неделю» —
  это взгляд от изменений в коде. Источник — коммиты и PR в репозиториях.
- **Вопросник заказчику** закрывает открытые вопросы из карты: задаём клиенту
  понятный выбор с рекомендацией, получаем ответ, обновляем базу знаний.

## Требования к сессии

- **project-status** и **client-questions** читают `docs/project-map.md` — запускать
  в сессии с доступом к репозиторию (GitHub MCP или локальная копия).
- **engineering-digest** работает в любой сессии с GitHub MCP (читает коммиты и PR).
