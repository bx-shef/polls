# Карта проекта — polls (сервис опросов ↔ Bitrix24)

> Точка входа в проект для людей и AI-агента. На этот файл опираются навыки
> отчётов (`/report-status`, `/report-questions`). Канон состояния — корневой
> `CLAUDE.md` и `docs/roadmap.md`; здесь — срез для отчётности.

## Цель

Сервис опросов (движок + версионирование + аналитика) в связке с Bitrix24:
переход сделки/лида в триггер-стадию → приглашение клиенту → прохождение опроса →
аналитика (NPS/CSAT) на дашборде и возврат результата в CRM. North-star —
датчик (CRM-событие) → AI → KPI/Лента.

## Что сделано

- Ядро (`src/`): domain (схема/метрики/ответы/компиляция/агрегация), store
  (`IStore`/`MemoryStore`/`PgStore`), api (`createApi` + анти-абьюз), obs (логгер/health).
- Bitrix24-ядро: crypto (AES-256-GCM), oauth/refresh, portal-стор, frame-handshake,
  deal/entity/task binding-слой, маппинг CRM→`CrmContext`, REST-клиент на `@bitrix24/b24jssdk`.
- Nuxt-приложение: контур A (`/s/:key`), дашборд (`/d/:key`), админ-UI (`/admin/surveys`),
  тёмная тема + тоггл, визуальный гейт (Playwright, 48 эталонов).
- Обновлены `@bitrix24/b24jssdk` 1.3→2.0 и `@bitrix24/b24ui-nuxt` 2.8→2.9 (сборка зелёная).
